# C/C++ 刷题模块需求说明

## 1. 模块定位

C/C++ 模块面向高校初中级学习者，用于课程复习、语言基础巩固和初级求职准备。模块负责从主页入口之后的完整练习闭环，公共主页、正式登录、用户中心和后端部署由团队公共层负责。

模块题库严格限定为 C/C++ 语言知识，不纳入 STL、数据结构、Linux、操作系统、计算机网络和考研专题。

## 2. 用户流程

1. 用户从团队主页进入 `/modules/cpp/pages/home/index`。
2. 未登录时跳转公共登录页；登录成功后返回原页面。
3. 用户选择章节练习、随机练习、错题本或收藏。
4. 开题前选择 5、10、20 题，默认 10 题；题量不足时取全部可用题。
5. 每题选择答案后主动确认。提交成功前保留选择，提交成功后不可修改。
6. 服务端返回正误、正确答案和解析；用户可独立切换收藏状态。
7. 最后一题完成后交卷，展示总分和分章节表现。
8. 答错题目自动进入错题本。错题模式答对后标记已掌握；以后再次答错则恢复未掌握。
9. 未完成会话保存在服务端，本地只缓存会话 ID 和当前题尚未提交的选择。

## 3. 页面与职责

- 模块首页：题库进度、累计答题、正确率、待掌握错题、收藏数、继续练习和四类入口。
- 章节页：9 个章节的题量、已练数量、进度和累计正确率。
- 练习设置页：展示模式与章节，选择 5/10/20 题并创建会话。
- 答题页：题目、代码、选项、提交状态、即时判题、解析、收藏和下一题。
- 结果页：总题数、正确数、错误数、正确率和章节表现。
- 错题本：全部、未掌握、已掌握筛选，查看解析并发起未掌握错题重练。
- 收藏页：查看解析、取消收藏并发起收藏重练。

## 4. 题库范围

首版共 100 道题，分布如下：

- C 基础与运算：12 题
- 流程控制与函数：11 题
- 数组与字符串：11 题
- 指针与动态内存：12 题
- 结构体、文件与预处理：11 题
- C++ 基础与引用：11 题
- 类与对象：11 题
- 继承与多态：11 题
- 运算符重载、模板与异常：10 题

题型包括 62 道单选、17 道多选和 21 道判断。所有题目有难度、标签、答案和解析；代码内容只读展示，不执行代码。

## 5. 业务规则

- 同一会话题目 ID 唯一，随机顺序由服务端决定，选项顺序保持题库顺序。
- 创建新会话时，旧的未完成会话标记为 abandoned，首页只展示一个可继续会话。
- 普通会话中答对不会自动删除历史错题；只有错题重做模式答对才标记 mastered。
- 任意模式再次答错已掌握题目时，重新标记为未掌握。
- 收藏与正误状态独立；取消收藏不会删除答题或错题记录。
- 进度为“至少作答过一次的不同题目数 / 当前有效题目总数”。
- 累计正确率为“所有成功提交中的正确次数 / 成功提交总次数”。
- 服务端不在待答题载荷中返回正确答案和解析。

## 6. 状态与错误处理

答题页状态依次为 loading、answering、submitting、reviewed 和 finished。提交失败时回到 answering，保留选择和 `clientAnswerId`，用户可安全重试。

- 401：清除失效 Token，跳转公共登录页并携带原页面地址。
- 超时或网络失败：显示可重试提示，不清空未提交选择。
- 5xx：显示服务异常并允许重试。
- 空题库：禁止进入答题页，明确提示当前没有可练题目。
- 已完成会话：直接跳转结果页；已废弃会话提示重新开题。

## 7. 验收标准

- 微信开发者工具成功编译，主包可进入 C/C++ 分包。
- Mock 和真实 API 使用相同页面与仓储接口。
- 单选、多选、判断均能完成选择、确认、判题和解析流程。
- 5/10/20 题、题量不足、随机无重复、恢复、交卷和重复提交符合规则。
- 错题掌握状态和收藏状态能够跨页面、跨小程序重启保持。
- 100 道题通过自动结构校验，并在发布前完成一名非出题人员的内容复核。
