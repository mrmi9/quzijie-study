# 趣字节刷题发布候选版清单

本清单以“微信正式版本可供真实用户使用”为完成标准。代码、测试和镜像门禁已经具备；外部资源配置完成前，体验版和正式版会被 `npm run verify:release` 阻止。

## 1. 当前必须由项目负责人配置

### 1.1 预发布服务器与域名

- [ ] 确定部署 API 的 Linux 服务器或云容器平台，并提供可执行 Docker/SSH 发布的权限。
- [ ] 准备一个已完成 ICP 备案的 API 子域名，例如 `api.<你的域名>`，完成 DNS 解析。
- [ ] 为该域名配置有效 HTTPS 证书，支持 TLS 1.2 及以上；公网只暴露 443，PostgreSQL 不直接暴露公网。
- [ ] 准备独立预发布 PostgreSQL 17 数据库、最小权限账号和备份存储位置。
- [ ] 不要在聊天或 Git 中发送 AppSecret、数据库密码、JWT 密钥或证书私钥；敏感值只写入服务器环境变量。

完成后只需告诉我：服务器选择、API 域名、是否已备案、是否允许我执行部署。密码或密钥由你在本机/服务器终端输入。

### 1.2 微信公众平台

- [ ] 确认 AppID 所属主体能够正式发布，项目成员具有开发、体验、审核和发布权限。
- [ ] 在“小程序后台 → 开发 → 开发设置 → 服务器域名”添加实际 HTTPS API 域名。
- [ ] 选择与刷题/学习内容相符的服务类目，并按公众平台当前页面准备所需资质。
- [ ] 在“账号设置 → 服务内容声明 → 用户隐私保护指引”填写与实际行为一致的说明。
- [ ] 提审版本再次核对隐私指引；平台隐私声明应覆盖微信登录标识、答题记录、错题、收藏和考试历史。

### 1.3 公开信息与内容复核

- [ ] 提供实际运营主体名称和公开的隐私联系渠道（建议专用邮箱），用于 `miniprogram/config/release.js` 和隐私页。
- [ ] 指定一名非出题人员完成 500 题逐学科交叉复核，并填写 `content/REVIEW_STATUS.md`。
- [ ] 确认小程序名称、简介、头像、类目、用户协议和隐私说明可以对外公开。

## 2. 由 Codex 在配置后继续完成

- [x] 准备不可变 API/迁移镜像发布工作流和生产专用 Compose。
- [x] 准备环境预检、自动迁移部署、健康检查、应用回滚、数据库备份/恢复脚本。
- [x] 准备通过发布门禁后调用微信开发者工具上传版本的脚本。
- [ ] 写入正式 API 域名、运营主体和隐私联系渠道，执行 `npm run verify:release`。
- [ ] 构建带 Git SHA 的 API/迁移镜像，部署预发布环境并执行迁移、500 题幂等导入。
- [ ] 验证 HTTPS、证书链、`/health`、`/ready`、日志脱敏、数据库备份与恢复。
- [ ] 在体验版完成真实登录、七学科普通练习、408、账户删除、弱网和恢复验收。
- [ ] 生成发布版本号与备注，通过微信开发者工具或 `miniprogram-ci` 上传代码。
- [ ] 陪同完成提交审核；根据审核反馈修复并重新上传。
- [ ] 审核通过后发布正式版本，执行上线观察和回滚演练。

## 3. 发布门禁

```powershell
npm run verify:all
npm run verify:release
docker compose -f compose.api.yaml config --quiet
docker compose -f compose.release.yaml config --quiet
```

`verify:release` 必须满足：正式 AppID、共享配置开启合法域名校验、HTTPS API 域名有效、运营者和隐私联系方式已填写、账户/隐私页面存在、500 题交叉复核记录已完成。

## 4. 发布完成定义

- 微信审核通过且正式版本已发布。
- 正式版连接生产 HTTPS API，不依赖 Mock、调试模式或“不校验合法域名”。
- 真实用户可以登录、完成普通练习和 408、恢复进度、查看错题收藏并删除账户。
- 监控、日志、备份、告警和回滚可用；上线观察期无阻断性问题。
