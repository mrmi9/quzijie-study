# 趣字节刷题全模块需求说明 v2

## 1. 产品定位与用户

趣字节刷题面向高校初中级学习者，用于课程复习、基础巩固、初级求职准备和考研 408 客观题训练。首版是原生微信小程序，不在客户端编译 C/C++，也不提供代码输入判题。

登录、正式用户资料和真实后端属于公共能力。本仓库用演示登录和 Mock Token 占位，未登录用户不能答题。

## 2. 首页模块与底层学科

首页固定显示五个产品入口：

- C/C++：直接进入 `cpp` 学科。
- Linux/操作系统：先选择 `linux` 或 `os` 方向。
- 数据结构：直接进入 `ds` 学科。
- 计网/STL：先选择 `network` 或 `stl` 方向。
- 考研 408：进入整卷客观题模拟，复用 `ds`、`co`、`os`、`network` 题目。

组成原理普通练习从 408 首页的“计算机组成原理专项练习”进入，因此五个首页主入口保持不变，同时七学科均可完成普通练习闭环。

七个底层学科的答题、进度、错题和收藏相互隔离；全局页面负责聚合。408 作答计入题目所属底层学科，不能把同一道题重复计入总进度。

## 3. 全局学习中心

原生 TabBar 包含“首页、错题、收藏”。首页展示今日答题数、累计正确率、500 题进度、未掌握错题数、收藏数、各模块进度、一个可继续的普通会话和一个可继续的 408 试卷。

全局进度计算公式为“至少答过一次的不同题目数 / 500 道有效题目”。全局错题和收藏支持全部及七学科筛选；从模块内部进入时固定当前学科。全局“全部”范围可跨学科重做所有未掌握错题或收藏题；跨学科随机和章节练习不允许创建。

## 4. 普通练习通用规则

七学科共用章节、随机、错题和收藏四种模式：

1. 可选 5、10、20 题，默认 10；候选题不足时取全部，单次会话不得重复。
2. 单选和判断只能保留一个选项；多选可反复点击选中或取消。
3. 未选择不能确认。确认后按题提交，成功前保留选择和 `clientAnswerId`，成功后答案锁定。
4. 每题提交成功后立即显示正误、正确答案和解析；收藏状态与正误独立。
5. 一组全部作答后才能完成，结果显示总题数、正确数、错误数、正确率和章节表现。
6. 同一用户只保留一个 active 普通会话；创建新会话时旧会话改为 `abandoned`。
7. 服务端保存会话；客户端只缓存未成功提交的当前选择。恢复时已提交题不可再次计分。

普通待答题载荷不得包含 `correctOptionIds` 或 `explanation`。题目支持只读代码和最多两张本地 PNG；代码块必须横向滚动，题图必须可预览且带 `alt` 替代说明。

## 5. 错题与收藏

- 任意普通模式或考试答错都会加入错题本；再次答错会累加次数并设为未掌握。
- 只有“错题重做”模式答对才标为已掌握，历史记录保留；已掌握题以后答错会恢复未掌握。
- 错题本支持全部、未掌握、已掌握筛选；掌握状态筛选只影响展示，不改变当前学科范围内的未掌握重做题池。
- 全局“全部”范围的错题与收藏重练均支持 5、10、20、全部四档并默认选择全部；全局错题只纳入有效且未掌握的题目。
- 收藏增删不改变答题和错题记录；收藏题可按单学科或全学科“全部”范围发起普通重练。

## 6. 题库范围与配额

运行时共有 500 道全局唯一题目：

- C/C++ 100：9 章，50 基础、35 进阶、15 难题。
- Linux 50：命令与 Shell 10、文件与权限 12、进程/任务/服务 10、用户/软件包/日志 8、网络排障 10。
- 操作系统 50：进程调度 10、同步死锁 10、内存 12、文件存储 10、I/O 安全 8。
- 数据结构 100：概论 10、线性表 14、栈队列 12、串数组 8、树 18、图 16、查找 10、排序 12。
- 计算机网络 50：体系结构 6、物理层 6、链路层 10、网络层 12、传输层 9、应用与安全 7。
- STL 50：泛型迭代器 8、顺序容器 12、关联容器 10、算法 10、适配器/函数对象/Lambda 10。
- 组成原理 100：数据运算 16、存储 18、指令 14、CPU/流水线 22、总线 8、I/O 14、汇编性能 8。

新增 100 题学科按 60 单选、20 多选、20 判断；50 题方向按 30/10/10。C/C++ 保留既有 62/17/21 题型结构。每个合计 100 题模块采用 50%/35%/15% 难度分布，两个 50 题方向合并后也满足该比例。

题目必填答案、解析、标签、难度、状态和版本。ID 使用 `cpp/lnx/os/ds/net/stl/co` 前缀并全局唯一。图资源单张不超过 100KB，总计不超过 600KB。

## 7. 408 客观题专项模拟

- 每卷固定 40 道单选题、60 分钟、每题 2 分、满分 80 分。
- 配比固定为数据结构 12、组成原理 12、操作系统 9、计算机网络 7；同卷不重复。
- 答题时不显示正误与解析；固定底栏持续提供上一题、答题卡和下一题，最后一题切换为检查并交卷，选择答案后停留当前题并允许修改。
- `expiresAt` 是唯一计时依据，切后台不暂停。草稿选择后保存，重启按实际剩余时间恢复。
- 提前交卷前提示未答数量；时间归零自动交卷，未答计错。
- 交卷后展示分数、正确率、四科表现和错题解析；错误题同步进入全局错题。
- 交卷接口幂等；已完成试卷不可修改。结果保存完整题目版本快照，题库更新不改变历史成绩。

## 8. 登录、网络与错误

- 所有业务仓储方法先检查 Token；401 清理 Token 并回到公共登录页，登录后返回原入口。
- 网络失败、超时和 5xx 显示可重试提示，不清除普通选择或考试本地草稿。
- 普通逐题提交依赖 `clientAnswerId` 幂等；试卷提交按试卷 ID 幂等。
- 空题库、题量不足、无错题或无收藏必须提供清晰状态，不能进入空白答题页。

## 9. 非功能与发布门禁

- 页面适配常见屏幕和安全区，不遮挡底部操作；代码横向滚动、题图可缩放。
- 主包与单个业务分包各使用 1.5MB 项目内预算，`npm run verify` 自动检查。
- Mock 与真实 API 共用页面与模型；页面不得直接调用 `wx.request`。
- 正式发布前需完成：合法 AppID、后端白名单、真实 API 联调、微信开发者工具与真机验收、非出题人员 500 题交叉复核。

## 10. 首版明确不做

在线编译、代码输入题、主观题自动评分、完整 150 分模拟卷、评论、社交分享、题目搜索、管理后台、游客数据合并和离线答题均不在当前范围。
