# 手工验收清单

## 环境与导航

- [ ] 微信开发者工具导入根目录后可以编译。
- [ ] 主包主页可以进入 C/C++ 分包。
- [ ] 清除本地缓存后进入模块会跳转登录页，演示登录后返回模块首页。
- [ ] 将仓储模式切换为 api 后，页面内没有 Mock 专用调用。

## 答题流程

- [ ] 单选和判断只能保持一个选项；多选可以选择和取消多个选项。
- [ ] 未选择时确认按钮禁用。
- [ ] 提交失败时选择仍保留，重试不会重复计分。
- [ ] 提交成功后无法修改选项，正确答案和解析显示正确。
- [ ] 代码块在窄屏上可横向滚动，不挤压选项布局。
- [ ] 最后一题可正常交卷并进入结果页。

## 题量、恢复与异常

- [ ] 章节和随机模式分别测试 5、10、20 题。
- [ ] 章节题量不足时使用全部题目且没有重复题。
- [ ] 答题中退出后，模块首页显示继续练习，恢复位置正确。
- [ ] 新建练习后旧会话不可继续。
- [ ] 模拟断网、超时、500 和 401，检查重试与登录回跳。

## 错题与收藏

- [ ] 普通模式答错后题目进入未掌握错题。
- [ ] 错题模式答对后变为已掌握且历史仍可查看。
- [ ] 已掌握题再次答错后恢复未掌握。
- [ ] 收藏和取消收藏能在答题页、收藏页和首页数量间同步。
- [ ] 错题或收藏不足所选题量时可正常使用全部可用题。

## 发布前内容复核

- [ ] 执行 `npm run verify` 全部通过。
- [ ] 非出题人员逐章抽查题干、标准答案、解析和代码标准。
- [ ] 对存在实现差异的题目确认已写明语言标准或前提。
- [ ] 复核人、日期和发现的问题记录在 `content/REVIEW_STATUS.md`。
